# Security

Please do not publish vulnerabilities, private journal content, credentials, purchase records, or other sensitive information in a public issue.

The current public release is local-first and has no account or payment backend. If you discover a security problem, open a minimal GitHub issue that requests a private contact channel without including exploit details.

Never commit signing keys, provisioning profiles, store credentials, API secrets, receipt-validation secrets, or service-account files to this repository.
